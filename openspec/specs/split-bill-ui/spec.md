### Requirement: 參與者輸入表單
系統 SHALL 提供網頁表單，讓使用者輸入幣別、參與者名稱與已付金額，並可動態新增或刪除參與者列。

#### Scenario: 初始畫面
- **WHEN** 使用者開啟首頁
- **THEN** 顯示至少一列空白參與者輸入欄（名稱 + 金額）與「新增參與者」按鈕

#### Scenario: 新增參與者
- **WHEN** 使用者點擊「新增參與者」
- **THEN** 表單新增一列空白輸入欄

#### Scenario: 刪除參與者
- **WHEN** 使用者點擊某列的刪除按鈕
- **THEN** 該列從表單中移除

### Requirement: 送出並顯示結算結果
系統 SHALL 將表單資料送至 POST /split，並將結果顯示在頁面上。

#### Scenario: 成功送出
- **WHEN** 使用者填寫至少一位參與者並點擊「計算」
- **THEN** 頁面顯示總金額、每人應分攤、每人淨額與建議轉帳清單

#### Scenario: 無需轉帳
- **WHEN** 所有人付款金額相同
- **THEN** 結果區顯示「無需轉帳」

#### Scenario: API 回傳錯誤
- **WHEN** 送出的資料不符合 API 規格（如金額為負）
- **THEN** 頁面顯示錯誤提示訊息
